# Укрупнённый план концепции создания и функционирования автоматизированной системы контроля работы проектной команды на базе методологии «ГосТех»
## 1.Введение

Настоящая концепция разработана в рамках выполнения выпускной квалификационной работы. Целью концепции является цифровая трансформация процессов управления проектной деятельностью, включая распределение задач, контроль исполнения и оценку эффективности работы команды, с использованием принципов и подходов методологии «ГосТех».

Предметная область — проектная команда — охватывает взаимодействие между участниками проекта, руководителями, заказчиками и экспертами. Концепция направлена на повышение прозрачности, эффективности и управляемости проектной деятельности, обеспечивающей сквозной контроль жизненного цикла задач и проектов.

## 2. Описание проблемы, цели, ограничений и приоритетных клиентских путей
### 2.1. Проблемы текущего состояния
- Фрагментированность процессов: отсутствие единой системы управления задачами и контроля сроков.
- Низкая автоматизация контроля: отсутствие инструментов оперативного отслеживания загрузки участников, рисков срыва сроков и качества выполнения работ.
- Отсутствие сквозной аналитики: невозможность объективной оценки вклада каждого участника и общей эффективности команды.
### 2.2. Цели концепции
1. Повысить доверие к результатам работы за счёт прозрачности распределения задач, фиксации изменений и верифицируемости статусов.
2. Создать единое цифровое пространство для всех участников проекта.
### 2.3. Ограничения
- Необходимость соблюдения требований ФЗ-152 при обработке персональных данных участников.
- Ограниченная цифровая зрелость пользователей (особенно в учебной среде).
- Требования к отечественности технологического стека и совместимости с принципами «ГосТех».
- Ограниченный срок реализации в рамках ВКР.
### 2.4. Приоритетные клиентские пути
- Участник проекта → получение задачи → выполнение → загрузка результата → получение обратной связи.
- Руководитель проекта → создание проекта → распределение задач → мониторинг прогресса → формирование отчётов.
- Заказчик / Научный руководитель → наблюдение за ходом реализации → проверка промежуточных результатов → оценка итогов.

## 3. Общее описание архитектуры
Архитектура системы реализуется в рамках домена «Управление проектной деятельностью» и строится по принципам доменной модели «ГосТех»:
- Функциональная архитектура: клиентоцентричные процессы, сгруппированные по сценариям жизненного цикла задачи («поставить задачу», «выполнить», «проверить», «закрыть»).
- ИТ-архитектура: микросервисная структура на базе платформы «ГосТех» и доменных сервисов («Управление задачами», «Аналитика проекта», «Отчётность»).
- Данные: единая модель данных, включающая сущности «Пользователь», «Проект», «Задача», «Роль», «Комментарий», «Статус выполнения», «Отчёт».
- Инфраструктура: размещение на отечественной облачной платформе (в рамках «ГосТех» или совместимой), обеспечение отказоустойчивости и масштабируемости.

## 4. Связность и интеграция
Концепция предусматривает интеграцию с внешними системами:

API для подключения внешних инструментов (Git, CI/CD, документооборот) — опционально.
Интеграция реализуются через стандартизированные API и соответствует требованиям «ГосТех» к интеграционной архитектуре.

## 5. Принципы проектирования
Проектирование архитектуры осуществляется на основе следующих принципов «ГосТех»:
- Клиентоцентричность: проектирование от реальных задач участников проекта и руководителей.
- Повторное использование: применение общих сервисов платформы «ГосТех».
- Масштабируемость и адаптируемость: поддержка роста числа проектов и пользователей, возможность настройки под разные типы проектов.
- Технологическая независимость: использование отечественного ПО и решений из реестра Минцифры России.

## 6. Информационная безопасность
Безопасность обеспечивается на всех уровнях:
- Персональные данные участников обрабатываются в соответствии с ФЗ-152.
- Все данные хранятся на территории РФ и защищаются сертифицированными средствами.
- Сервисы разрабатываются по методологии безопасного программного обеспечения (secure SDLC).
- Инфраструктура соответствует требованиям ФСТЭК и ФСБ РФ, включая возможность последующей аттестации.

## 7. Экономическая эффективность
Реализация концепции обеспечивает следующие эффекты:
1) Снижение временных затрат на управление проектом (до 60%) за счёт автоматизации.
2) Повышение качества контроля — исключение «человеческого фактора» при отслеживании сроков и статусов.
3) Увеличение удовлетворённости пользователей за счёт прозрачности, удобства интерфейса и своевременных уведомлений.
3) Снижение административной нагрузки на руководителей и кураторов.

##
Концепция соответствует требованиям методологии «ГосТех» и может быть реализована в рамках дорожной карты создания доменной архитектуры цифровой образовательной или управленческой среды.
